# Contributing to RabbitLens

First off, thank you for considering contributing to RabbitLens! It's people like you that make open-source a great community.

## Development Workflow

1. **Fork & Clone**: Fork the repository and clone it locally.
2. **Start Backend**: Run `make up` to spin up a local RabbitMQ instance via Docker.
3. **Start Frontend**: Run `make dev` to start the Vite dev server with Hot Module Replacement (HMR).
4. **Code**: Make your brilliant changes.
5. **Test**: Ensure everything works as expected.

## Pull Request Guidelines

- Create a new branch for your feature (`feature/amazing-thing`) or bugfix (`fix/nasty-bug`).
- Keep your changes focused and atomic.
- Write clear, concise commit messages.
- Ensure the UI looks perfect in both Light and Dark modes.
- Open a Pull Request and describe the problem you are solving.

Let's build the best RabbitMQ UI together!
